Bedienung

Hier wird beschrieben wie man neue Rezepte und andere Daten in den CManager eingibt.

Grundsätzliches

Viele die Daten betreffenden Funktionen sind im CManager über das Kontextmenü erreichbar. Über das Kontextmenü im Rezeptebaum können neue Einträge angelegt, vorhandene gelöscht und bearbeitet werden. Dies gescheiht stets unabhängig davon um welche Art von Daten es sich dabei handelt. D.h. ein neues Glas wird in gleicher Weise angelegt wie ein neues Rezept.

Beachten Sie bei allen Datenänderungen, dass die Daten sich aufeinander beziehen. Wenn Sie z.B. ein Glas löschen, das bei einigen Cocktails als Standardglas hinterlegt ist, geht dieser Bezug verloren. Die betreffenden Rezepte haben dann kein standardglas mehr und müssen ggf. einzeln korrigiert werden.

Machen Sie daher gelegentlich Sicherungskopien in Form von vollständigen Exporten (im Hauptmenü unter "Datei"). Wenn Sie versehentlich ewtas geändert haben, können Sie dann durch einen Import dieser Sicherungsdatei die Daten wieder herstellen, und damit auch evtl. zerstörte Bezüge wieder korrigieren.

Neue Daten anlegen

Selektieren Sie eine Datenüberschrift im Rezeptebaum und aktivieren Sie den Menüpunkt "Neu" im Kontextmenü. Es wird daraufhin ein zur Überschrift untergeordnetes Element neu angelegt. Dieses erhält den Namen "* Neues ...", wobei die drei Punkte durch die Art des neu Erzeugten Datensatzes ergänzt werden. Also "* Neues Glas" für ein neues Glas.

Bei Untergeordneten Datenelementen, wie z.B. einer Zutatenzuordnung in der Zutatenliste eines Cocktails wählen Sie stattdessen den Button "*" über der betreffenden Liste. Auch hier wird ein neues Element (hier eine Zuordnung) angelegt, die mit einem voreingestellten Namen für neue Elemente vorbelegt ist.

Alle neu angelegten Daten werden sofort in der Datenbank gespeichert und finden sich auch nach dem Verlassen von CManager nach einem erneuten Start wieder.

Daten bearbeiten

Das Bearbeiten der Daten erfolgt jeweils in speziellen Bearbeitungsdialogen, die abhängig von der Datenart unterschiedliche Eingabefelder oder Auswahllisten enthalten. Diese folgen jedoch alle einem einheitlichen Schema.

Um einen Datensatz (z.B.eine neue Zutat) zu bearbeiten selektieren Sie  diesen im Rezeptebaum oder in der Liste untegeordneter Elemente eines Bearbeitungsdialoges. Wählen Sie dann im Kontexmenü des Rezeptebaums den Punkt "Bearbeiten" oder benutzen Sie den ">"-Button über der Liste untergeordneter Elemente. In jedem Fall wird dann der Bearbeitungsdialog für die betreffende Datenart geöffnet und mit den Daten des zuvor selektierten Elementes gefüllt.

Häufig finden Sie als erstes bzw. oberstes Element im Bearbeitungsdialog eine sog. Combobox. Dies ist eine aufklappbare Liste deren aktuelle Auswahl in einem Textfeld dargstellt wird. Das Aufklappen erfolgt über einen kleinen Button rechts neben dem Textfeld. Über diese oberste Combobox können Sie die Zuordnung zu einer übergeordneten Datenart vornehmen. So sind z.B. die Rezepte jeweils einer Rubrik zugeordnet, die sie über diese Combobox verändern können.

Meist befindet sich erst unter diese Combobox ein Textfeld über das der Name des Elementes bearbeitet werden kann. Grundsätzlich finden Sie Comboboxen wenn andere Elemente dem gerade bearbeiten Element zugeordnet werden können. So kann z.B. das Standardglas in dem der Cocktail serviert dem Cocktailrezept über eine Combobox zugeordnet werden.

Alle Bearbeitungsdialoge enthalten einen OK-Button über den die Dateneingabe beendet wird und dabei die Daten wieder in der Datenbank abgelegt werden. Mit dem ebenfalls stets vorhandenem Abbruch-Button können Sie hingegen den Bearbeitungsdialog verlassen, ohne dass Daten gespeichert werden.

Daten löschen 

Das Löschen von Daten ist einfach und schnell möglich. Achtung - wenn die Daten einmal gelöscht wurden können Sie nicht wieder hergestellt werden.

Um ein Element zu löschen selektieren Sie dieses im Rezeptebaum und wählen Sie die Funktion "Löschen" des Kontextmenüs. Wenn Sie untergeordnete Daten löschen wollen müssen Sie das übergeordnete Element zur Bearbeitung öffnen. Markieren Sie nun das untergeordnete Element, z.B. eine Zutatenzuordnung, in der Liste der untergeordneten Elemente. Löschen Sie das Element durch Betätigung des "X"-Buttons über der betreffenden Liste. 

Wenn Sie die Sicherheitsrückfrage mit "Ja" beantworten wird das Element unwiederruflich gelöscht.

Machen Sie häufig Backups, damit Sie auch in solchen Fällen die Daten ggf. wieder herstellen können. Nutzen Sie dazu die Export-Funktion bzw. die Import-Funktion von CManager. Achtung - ein Import bringt alle seit dem Export veränderten Elemente wieder auf den Stand des Exports zurück. Lediglich neu erfasste Daten bleiben erhalten. Häufig ist die Neuerfassung einiger verloren gegangender Änderungen das kleinere Übel.
